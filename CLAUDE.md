# Hola — Project Guide for AI Assistants

Dating/social app frontend. Mobile-first, single-page, connects to a Metarhia REST backend.

## Stack

| Layer | Tech |
|---|---|
| Framework | TanStack Start (React 19) |
| Router | TanStack Router v1 (file-based) |
| Data fetching | TanStack Query v5 + ky v1 |
| Validation | **Zod v4** |
| Build | Vite 7, Bun |
| Testing | Vitest 4 + Testing Library |
| Styling | CSS Modules |

## Zod v4 — Critical Differences

Zod v4 removes the `z.*` namespace pattern:

```ts
// WRONG (Zod v3)
import { z } from 'zod'
z.object({ name: z.string() })
z.infer<typeof schema>

// CORRECT (Zod v4)
import { object, string, output } from 'zod'
object({ name: string() })
output<typeof schema>
```

## Project Structure

```
src/
  app/          # providers, layout (AppShell, TopBar, BottomNav)
  routes/       # file-based routes — (app)/* and (auth)/*
  pages/        # page components (one per route)
  features/     # self-contained feature slices (Atomic Design)
  entities/     # domain models (user, match, chat, …)
  widgets/      # composite UI blocks
  shared/       # api client (ky), utils, types
```

- Path alias: `@/*` → `src/*`
- Route tree is auto-generated: `src/routeTree.gen.ts` — do not edit manually

## Route Layout

```
/               → index (landing / redirect)
/(auth)/login
/(auth)/signup
/(app)/feed
/(app)/matches
/(app)/chats
/(app)/chat/$chatId
/(app)/profile          ← layout route with <Outlet />
/(app)/profile/         ← ProfilePage (index)
/(app)/profile/edit     ← ProfileEditPage
/(app)/settings
```

Auth guard is in `beforeLoad` of each `(app)` route — reads `getAccessToken()` and redirects to `/login` if missing.

## AppShell Props

```ts
<AppShell
  title="Page Title"
  showBottomNav={true}   // default true
  scrollable={false}     // default false — enables scroll on main
  onBack={() => navigate(-1)}  // shows back arrow in TopBar
>
```

## Commands

```bash
bun run dev          # start dev server on :3000
bun run build        # production build
bun run test:run     # run tests once
bun run lint         # ESLint
bun run format       # Prettier
npx tsc --noEmit     # type check
```

Backend types are auto-generated from `../backend` before dev/build.

## Backend

Metarhia REST API at `../backend/`. Auth endpoints: `POST /api/auth/login`, `POST /api/auth/signup`, `POST /api/auth/logout`. Token stored in localStorage via `shared/api/http.ts`.

## Conventions

- CSS Modules only — no inline styles, no Tailwind
- No barrel `index.ts` re-exports unless the folder is a public API boundary
- Keep pages thin — business logic goes into features or entities
- `routeTree.gen.ts` is auto-generated by `@tanstack/router-plugin` on every `bun run dev` — never edit it
